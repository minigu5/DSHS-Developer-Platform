# 바이브 코딩 가이드 작성법

이 폴더(`src/content/guides/`)에 있는 `.md` 파일이 곧 가이드 화면에 표시되는 내용입니다.
파일을 추가하거나 수정하면 바로 반영돼요. **이 `README.md` 자체는 가이드로 노출되지 않습니다.**

> 작성에 코드를 건드릴 필요는 없어요. 마크다운 파일만 만들면 됩니다.

---

## 0. 가장 빠른 시작 — 템플릿 복붙

아래 내용을 그대로 복사해 새 `.md` 파일로 저장하면 3단계짜리 가이드가 됩니다.

```markdown
# 가이드 제목

이 가이드가 무엇을 도와주는지 한두 문장으로 소개해요.

---

# 1단계: 무엇을 할지

첫 번째로 할 일을 설명해요.

---

# 2단계: 직접 해보기

```bash
echo "터미널 명령은 이렇게 코드블록으로"
```

---

# 3단계: 마무리

끝! 완성한 걸 친구들에게 공유해보세요.
```

> 줄 하나에 `---` 만 있으면 그 지점에서 **단계가 나뉩니다.** (이전/다음으로 넘겨 봄)

---

## 1. 파일 이름 = "어떤 선택 조합에 보여줄지"

사용자가 위저드에서 고른 값들을 `--` 로 이어서 파일 이름을 짓습니다.
**모든 항목을 다 넣을 필요는 없어요** — 의미 있는 항목만 골라 넣으면 됩니다.

### 사용할 수 있는 값

| 항목 | 키 | 값 |
|---|---|---|
| 프로그램 종류 | `type` | `website`, `app`, `extension`, `cli`, `library` |
| 앱 플랫폼 (종류가 `app` 일 때만) | `appPlatform` | `android`, `ios`, `ipados`, `macos`, `windows` |
| 터미널 사용 | `terminal` | `terminal`(사용), `gui`(미사용) |
| AI 도구 | `ai` | `chatgpt-codex`, `claude-code`, `gemini` |
| 운영체제 | `os` | `windows`, `mac`, `linux` |

### 파일 이름 예시

| 파일 이름 | 언제 보이나 |
|---|---|
| `website--claude-code--mac.md` | 웹사이트 + Claude Code + macOS |
| `claude-code--mac.md` | Claude Code + macOS (프로그램 종류 무관) |
| `claude-code.md` | Claude Code 를 고른 모든 경우 |
| `website.md` | 웹사이트를 고른 모든 경우 |
| `default.md` | 위에서 아무것도 안 맞을 때 (최종 폴백) |

> ⚠️ 값과 구분자(`--`)의 **철자·하이픈을 정확히** 맞춰야 합니다. (`claude-code` ⭕ / `claudecode` ❌)

## 2. 매칭 우선순위 (부분 매칭 + 폴백)

한 번의 선택에 대해, 아래 **순서대로** 파일을 찾아보고 **가장 먼저 발견된 파일** 하나를 보여줍니다.
즉 구체적인 파일이 일반적인 파일보다 우선합니다.

```
1) type + appPlatform + terminal + ai + os   (가장 구체적)
2) type + appPlatform + ai + os
3) type + ai + os
4) ai + os
5) type + ai
6) ai
7) type
8) default                                    (최종 폴백)
```

(앱이 아니면 `appPlatform` 이 없으니 1·2번은 자동으로 건너뜁니다.)

### 실제로 어떻게 골라지는지 — 예시

사용자가 **웹사이트 / 터미널 사용 / Claude Code / macOS** 를 골랐다고 하면,
시스템은 이런 순서로 파일을 찾습니다:

```
website--terminal--claude-code--mac.md   ← 없음
website--claude-code--mac.md             ← 없음
claude-code--mac.md                      ← 있음! → 이 파일을 보여줌 ✅
```

따라서 **모든 조합마다 파일을 만들 필요가 전혀 없습니다.**
- 도구별로 통일된 안내면 → `claude-code.md`, `gemini.md` 처럼 `ai` 만으로 충분
- OS 차이가 중요하면 → `claude-code--mac.md`, `claude-code--windows.md` 로 쪼개기
- 아주 특수한 조합만 → 긴 이름의 파일로 예외 처리
- 그 외 전부 → `default.md` 가 받아줌

## 3. 단계 나누기 — `---`

파일 안에서 **줄 하나에 `---` 만** 적으면 그 지점에서 단계가 나뉩니다.
각 단계는 가이드 화면에서 "이전 / 다음" 버튼으로 하나씩 넘겨 봅니다.

```markdown
# 1단계
첫 화면 내용

---

# 2단계
두 번째 화면 내용
```

> 단계 안에서 가로줄을 그리고 싶다면 `---` 대신 `***` 를 쓰세요. (`---` 는 단계 구분 전용)

---

## 3-1. 마지막 "다음으로 해볼 것" 카드 — `===`

가이드 **맨 끝**에 줄 하나로 `===` (등호 3개 이상) 를 넣으면, 그 아래는 단계가 아니라
**다음 행동 카드**가 됩니다. 마지막 단계에 도달하면 화면이 이 카드들로 부드럽게 스크롤돼요.

```markdown
# 마지막 단계
가이드 본문...

===

# 다음으로 해볼 것

- [배포 가이드로 이동](/guide/website/terminal/claude-code/mac) — 웹사이트가 완성됐다면, 인터넷에 배포해보세요.
- [팁 둘러보기](/tips) — 다른 친구들의 노하우를 구경해보세요.
- [공식 문서](https://example.com) — 더 깊이 알고 싶다면.
```

규칙:
- `===` 위쪽은 평소처럼 `---` 로 단계를 나눕니다. `===` 아래쪽이 카드 영역입니다.
- 맨 위 `# 제목` 한 줄은 카드 묶음의 제목이 됩니다(생략하면 "다음으로 해볼 것").
- 카드 한 개 = 목록 한 줄: `- [버튼라벨](링크) — 설명문`
  - `[버튼라벨](링크)` → 카드 아래쪽 버튼 (라벨/링크)
  - `—`(또는 `-`, `:`) 뒤의 글 → 카드 설명문
- 링크가 `/` 로 시작하면 사이트 내부 이동, `http`로 시작하면 새 탭으로 열립니다.

## 4. 쓸 수 있는 마크다운 문법 (치트시트)

````markdown
# 큰 제목        ## 중간 제목        ### 작은 제목

**굵게**, *기울임*, `인라인 코드`

- 순서 없는 목록
- 또 다른 항목

1. 순서 있는 목록
2. 두 번째

> 인용문 — 팁이나 주의사항에 좋아요

[링크 텍스트](https://example.com)   ← 새 탭으로 열림

![이미지 설명](https://i.ibb.co/xxxx/screenshot.png)

```bash
# 코드블록 (언어 이름을 붙이면 좋아요: bash, ts, python ...)
npm install -g @anthropic-ai/claude-code
```

| 표 | 헤더 |
|---|---|
| 칸 | 칸 |
````

렌더링 동작:
- **링크**는 항상 새 탭으로 열립니다.
- **이미지**는 외부 URL(예: imgbb) 을 그대로 넣으면 됩니다.
- **표·체크리스트** 등 GitHub 스타일 문법(GFM)을 지원합니다.

---

## 5. 작성 팁

- 한 단계에 **한 가지 일**만 담으면 따라 하기 쉬워요.
- 명령어·코드는 꼭 코드블록(```` ``` ````)으로 감싸 복사하기 좋게 만드세요.
- 스크린샷이 있으면 이해가 빨라집니다. imgbb 등에 올리고 이미지 URL 을 넣으세요.
- 먼저 `default.md` 같은 일반 가이드를 채우고, 필요할 때 구체적인 파일을 추가하는 순서를 추천해요.

## 6. 현재 들어있는 예시 파일

- `default.md` — 어떤 조합에도 맞는 게 없을 때의 기본 가이드
- `claude-code--mac.md` — 작성 형식을 보여주는 예시 (Claude Code + macOS)

위 파일들을 열어 형식을 참고하고, 자유롭게 복사·수정해서 새 가이드를 만들어 보세요.
